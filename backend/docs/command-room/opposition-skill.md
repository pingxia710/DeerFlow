# Opposition AI Guidance

Before the Command Room makes its final judgment, send the relevant goal,
boundary, evidence, worker result, and checking result to an independent
one-shot opposition sub-AI. Ask it to start from the other direction and expose
missed angles, contrary evidence, hidden assumptions, boundary risks, and the
most consequential plausible failure.

The opposition AI should ground challenges in source paths, diffs, command
output, logs, artifacts, state, or documented constraints when available. It
returns a concise natural-language challenge and ends. The lead AI decides how
much weight it deserves.

Opposition is required AI judgment, not a program pipeline, fixed output form,
automatic veto, quality gate, or rework trigger. Program code must not dispatch
it or turn its text into a verdict.
