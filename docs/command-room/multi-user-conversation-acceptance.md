# Multi-user conversation acceptance

Acceptance for multi-user work is task-specific: the requested result should be available to the authorized user, and any relevant permission boundary must be respected.

Use targeted observations—such as a visible result, a command outcome, or an access check—when they are relevant to the question being answered. Program code must preserve hard authorization boundaries without inferring a quality verdict or forcing a role sequence. The Command Room AI itself must delegate execution, have a different AI check each result, obtain an independent opposition view, and make the final decision without requiring a fixed form or governance record.
