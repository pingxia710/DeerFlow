# Command Room principles

Command Room is the continuous lead brain. It keeps the user's goal, progress,
context, boundaries, and final judgment while one-shot professional sub-AIs
execute bounded work.

- Do not manufacture workflow for conversation or a direct Chair answer.
- An explicit `work_package_id` isolates Context, plans, delivery cycles,
  governance records, and factual receipt ledgers. A package never overlaps its
  own Context/Planning/Technical Design with Execution.
- Context is optional. When it starts, bounded discovery handoffs may run in
  parallel, then a Recorder preserves one factual `00-context/context.md`
  snapshot before Planning or Technical Design begins.
- Planning and Technical Design are optional. When used, independent forward
  and opposition AIs start from the same Context snapshot and return separate
  natural results; the Chair forms the one accepted decision and receives human
  confirmation before Execution.
- Every real execution cycle is followed by a different AI review cycle against
  the actual result. Multiple Chair-declared Execution N tasks may run in
  parallel; Review N starts only when all admitted execution tasks in that
  package/cycle are terminal. Review depth follows the deliverable: simple
  factual work gets a simple check; code or high-risk work gets semantic,
  behavioral, regression, and boundary checks where relevant.
- A confirmed package N may execute in parallel with Context/Planning for an
  independent package N+1 only when the Chair declares separate scopes and
  owned paths.
- Markdown files and the real workspace carry shared AI state. Children do not
  share chat context, remain resident, or launch each other.
- Runtime may enforce explicit order, cycle identity, artifact receipt, paths,
  hashes, and permissions. It must not interpret findings, choose roles, decide
  whether to continue, judge quality, require rework, or declare completion.
- The Chair reads complete AI-authored text and explicitly decides whether to
  finish, begin a fresh Command Room run for re-planning/design, request
  authorization, or start the next execution cycle.
- Pause before production/public actions, credentials, money, customer data,
  destructive or irreversible operations, or expanded permission.
- Keep durable rules short and failure-driven. Do not promote temporary task
  discussion into project policy without an explicit decision.
