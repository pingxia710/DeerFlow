# Command Room run guidance

Use the smallest amount of coordination that serves the task.

- A simple local change can be inspected, changed, and checked by the lead AI alone.
- Ask a focused helper to investigate, challenge, or verify only when that creates useful independent signal.
- Keep shared notes only when they make an ongoing task clearer; their names and format are unconstrained.
- Treat command output, diffs, files, logs, and observed behavior as stronger than an unverified summary, while still using judgment about relevance and freshness.
- A check is worthwhile when its result answers a concrete question and can change the next action. Do not require a check, a role, or a report solely because a process document says so.

Hard boundary: do not perform or authorize production/public operations, use credentials, handle funds or customer data, make destructive/irreversible changes, or expand permissions without the appropriate user authorization. Record runtime facts and enforce permissions without turning runtime into a quality judge or task controller.
