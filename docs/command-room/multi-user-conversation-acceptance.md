# Multi-user conversation acceptance

Acceptance for multi-user work is task-specific: the requested result should be available to the authorized user, and any relevant permission boundary must be respected.

Use targeted observations—such as a visible result, a command outcome, or an access check—only when they are relevant to the question being answered. The system must preserve hard authorization boundaries and must not infer a quality verdict, force a review sequence, or require a fixed collection of governance records.
