# Multi-user conversation acceptance

Acceptance is task-specific: the requested result must be available to the
authorized user and relevant permission boundaries must hold.

Conversation and clear direct answers use no workflow. Every delegated real
action receives an independent same-cycle Review against the actual result.
Planning and Technical Design are optional and used only when their decisions
are genuinely needed. Program code preserves authorization and structural
stage/cycle/artifact facts; it does not infer quality, choose roles, advance the
work, or trigger rework. The Chair decides from complete natural-language
returns.
