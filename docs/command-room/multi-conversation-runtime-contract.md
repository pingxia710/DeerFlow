# Multi-conversation runtime contract

A multi-conversation runtime may associate actions, outputs, artifacts, and observed events with their parent execution context. It may support parallel work and preserve references needed for traceability.

It records facts and enforces hard access or permission limits. It does not evaluate answer quality, automatically route work to a role, initiate rework, or determine completion. The Command Room AI, not the runtime program, delegates execution, passes every worker result to a different checking AI, obtains an independent opposition result, and makes the final judgment from their natural-language returns.
